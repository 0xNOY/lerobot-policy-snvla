## Future-run training safeguards (2026-07-16)

These changes are for a **fresh future run**. They do not alter or stop the production process that
launched on 2026-07-15, and must not be injected into that run or one of its resumes.

- Future datasets use one uniform forward-only narration window of `20` frames. Do not introduce
  event-name-specific window sizes: that would couple augmentation to the current pick/place
  vocabulary and reduce transferability. With `n_action_steps=10`, a 20-frame target interval
  covers up to two post-event chunk-boundary narration decisions; neighboring narration centers
  remain midpoint-clipped by the generic augmentation algorithm.
- Future collection emits a completed result and the next-action preview as one target on the same
  simulator-event frame, for example ` (done)\nPutting ...`. The first pick preview remains the
  episode's initial narration, the final place emits ` (done)\n` alone, and `Task completed.\n`
  remains a separate target only after fixed-home arrival. Concatenating all emitted targets still
  produces the same canonical text stream. The validator accepts both the historical split format
  and this future combined format so immutable datasets remain auditable.
- Use `--policy.max_text_loss_tokens=24` for the combined targets. With the current PaliGemma
  tokenizer, the longest T1 combined body is 16 tokens and the complete BON/body/EOS/BOA target is
  19 loss-bearing tokens; the historical value 16 would silently omit the target suffix.
- The SNVLA FSDP entrypoint supports an opt-in safe-step checkpoint signal. Launch with
  `--checkpoint-on-signal=SIGUSR1`; rank 0 atomically publishes
  `<output_dir>/signal_checkpoint_rank0.json`. Send `SIGUSR1` only to the worker PID in that file,
  not to the `accelerate` launcher or process group. The request is max-reduced on every rank after
  an optimizer update, then the existing full model/optimizer/scheduler checkpoint path runs. The
  original `save_freq` is restored before `train_config.json` is serialized. Wait for `last` to
  point to a complete new checkpoint before sending a termination signal.
- Observation-noise defaults are now scale `0.0..0.025` with normalized Gaussian samples capped at
  `±2σ`. State outliers are no longer clamped to `[-1,1]` after noise, avoiding a discontinuous snap
  to q01/q99. On the actual v7 statistics, the per-axis worst-case xyz differences are bounded by
  `7.43/13.55/9.58 mm`; a deterministic 2,000-frame audit measured position delta median/p95/max
  `7.89/14.38/17.86 mm` and relative rotation delta `3.04/8.79/9.01°`. Local ignored evidence is
  `outputs/noise/max_state_noise_scale_0p025.json`.
- Enable total-duration LR fitting with `--policy.scheduler_auto_steps_enabled=true`. Defaults retain
  the PI schedule's relative shape: warmup ratio `1/30`, decay ratio `1.0`, and final/peak LR ratio
  `0.1`. For 359552 steps at peak `1e-4`, this resolves to warmup `11986`, decay `359552`, and final
  LR `1e-5`, instead of reaching the LR floor at step 30000. The ratios are configurable through
  `scheduler_warmup_ratio`, `scheduler_decay_ratio`, and `scheduler_final_lr_ratio`. A resume is
  rejected if its saved absolute schedule does not match the newly derived curve.
